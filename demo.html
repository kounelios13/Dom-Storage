Dom Storage

Dom Storage is a simple way to cache jQuery objects into memory.It is based on the following article

Dom Storage has the following methods for retrieving and recaching a jQuery object:

get(selector)

Returns the jQuery object that corresponds to that selector.To do this Dom Storage first looks through its internal cache for a reference to that object. If there is no refeerence it create a new jQuery object that corresponds to the selector,caches it and the returns it.
.Example:
var domStorage = new DomStorage();
domStorage.get("#foo p").css("text-align","left"); It will return an jQuery object that corresponds to #foo p selector.

update(selectors)

Updates the jQuery object that correspond to a given selector.When is this needed? Consider the previous example from get() method.What happens if new p are added on the fly inside foo?If you try to do the following: domStorage.get("#foo p").css("text-align","left"); you will notice that the new paragraphs won't be styled.This is because the object retrieved from get() reflects doesn't reflect the new changes that happened.To avoid this you can do the following before trying get():
domStorage.update("#foo p").This will recache the selector to reflect the new changes made on the fly Warning!!!.If a given selector does not exist in the cache a type error will be thrown.

getCache()

Returns the cache of a DomStorage object
